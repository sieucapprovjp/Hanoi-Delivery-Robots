---
1. recordRoadExperience: Updates road condition memory based on traversal speed.
---
[SIGNATURE]
- Input     : fromLat: float # Starting latitude
- Input     : fromLon: float # Starting longitude
- Input     : toLat: float # Ending latitude
- Input     : toLon: float # Ending longitude
- Input     : speedMultiplier: float # Observed speed ratio
- Output    : void
- Calls     : toFixed()
- Constants : CONFIG.ROBOT.EXPERIENCE_PENALTIES, CONFIG.ROBOT.EXPERIENCE_THRESHOLDS, CONFIG.ROBOT.MEMORY_UPDATE_WEIGHT
- Formulas  : key = string interpolation of coordinates; penalty = ternary check against thresholds; weighted average = current * weight + observed * (1 - weight)

[LOGIC]
1. Generate string key from truncated coordinates (4 decimal places).
2. Determine penalty level (heavy/moderate/light) by comparing speedMultiplier to predefined thresholds.
3. Apply exponential moving average to update stored penalty for given road segment.

---
2. decayMemory: Gradually resets road penalties toward baseline.
---
[SIGNATURE]
- Input     : None
- Output    : void
- Calls     : Math.abs()
- Constants : CONFIG.ROBOT.MEMORY_CLEANUP_THRESHOLD, this.memoryDecay
- Formulas  : penalty = penalty * decay + 1.0 * (1 - decay)

[LOGIC]
1. Iterate over all keys in roadMemory.
2. Pull penalty values toward 1.0 using decay factor.
3. Delete keys if penalty is sufficiently close to 1.0 based on cleanup threshold.

3. update: Main simulation step for movement and state management.
---
[SIGNATURE]
- Input     : None
- Output    : void
- Calls     : arriveAtWaypoint(), distanceTo(), mapManager.getRainPenaltyAt(), mapManager.getTrafficAt(), Math.max(), recordRoadExperience(), decayMemory(), maybeReroute(), marker.setLatLng(), marker.isPopupOpen(), updatePopup(), goCharge(), startCharging()
- Constants : CONFIG.ROBOT.STATUSES, CONFIG.ROBOT.METERS_PER_DEGREE, CONFIG.ROBOT.MIN_SPEED_MULTIPLIER, CONFIG.ROBOT.FRAME_COUNT_RECORD_MEMORY, CONFIG.ROBOT.BATTERY_LOW_THRESHOLD, CONFIG.ROBOT.CHARGING_ARRIVAL_THRESHOLD
- Formulas  : ratio = (speed * multiplier) / dist; lat/lon interpolation; battery -= drain * rainPenalty; speedMultiplier = max(MIN, (BASE * (1 - traffic * IMPACT)) / rainPenalty)

[LOGIC]
1. Exit if not in MOVING status.
2. Advance pathIndex or call arriveAtWaypoint if at segment end or path exhausted.
3. Snap to target if within movement range; otherwise interpolate position.
4. Update total distance traveled in meters.
5. Update battery levels considering local rain intensity.
6. Recalculate speedMultiplier based on traffic congestion and rain impact.
7. Periodically record experience and decay memory based on frame count.
8. Trigger rerouting or charging if battery/traffic thresholds met.
9. Sync UI marker position and refresh popup if open.

- Calls     : Math.max()
- Constants : CONFIG.ROBOT.BATTERY_PROJECTED_DRAIN_FACTOR, CONFIG.ROBOT.BATTERY_SAFETY_MARGIN
